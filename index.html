    Frequently Asked Questions About PDF Merging

    Client-side processing means your PDF files are combined entirely within your web browser using JavaScript and WebAssembly. The complete merging operation happens on your computer without any part of your documents being transmitted over the internet or stored on remote servers. This architecture provides three key security benefits:

    1. No Data Transmission: Your files never leave your device during the merging process, eliminating interception risks.
    2. Automatic Cleanup: All document data is immediately purged from memory when you close the tab or complete the operation.
    3. No Residual Files: Unlike desktop software, we don't create temporary files that might remain on your system.

    Yugal PDF Merger supports virtually all PDF formats and variants, including:

    • Standard PDFs: All versions from PDF 1.3 through PDF 2.0 (ISO 32000-1/2 compliant)
    • Password-Protected Files: Encrypted PDFs with 40-bit and 128-bit RC4 encryption
    • Complex Documents: PDFs containing images, vector graphics, and embedded fonts
    • Interactive PDFs: Forms, annotations, and digital signatures (preserved but not editable)
    • Scanned Documents: Image-based PDFs from scanners or mobile apps

    The only limitation is that password-protected PDFs will be merged but remain password-protected in the output document.

    For optimal performance when merging large PDF documents (500+ pages or 100MB+), follow these recommendations:

    • Use Chrome or Firefox: These browsers generally handle large files better than Safari or Edge
    • Close Other Tabs: Free up system resources by closing unnecessary browser tabs
    • Restart Browser: Start with a fresh browser session to clear memory
    • Update Browser: Ensure you're using the latest version of your web browser
    • Split Very Large Jobs: For documents over 1GB, consider merging in batches
    • Use a Powerful Device: More RAM and faster processors will improve performance

    Remember that processing time scales with document complexity more than pure file size - a 100MB scanned document may process faster than a 50MB PDF with complex vector graphics.
